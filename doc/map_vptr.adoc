
## vptr_map

### Synopsis

```c++
namespace boost {
namespace openmethod {
namespace policies {

### Synopsis

template<
    class Policy, class Base = extern_vptr,
    class Map = std::unordered_map<
        type_id,
        std::conditional_t<
            std::is_base_of_v<indirect_extern_vptr, Base>, const vptr_type*,
            vptr_type>>>
class vptr_map : virtual Base {
    static Map vptrs;

    template<typename ForwardIterator>
    static auto register_vptrs(ForwardIterator first, ForwardIterator last) -> void;

    template<class Class>
    static auto dynamic_vptr(const Class& arg) -> const vptr_type&;
};

}}}
```

### Description

`vptr_map` is an implementation of `external_vptr that stores the pointers to
the v-tables in a map.

`Policy` is the policy containing the facet. `Base` must be either `extern_vptr`
or `indirect_external_vptr`. `Map` is an `AssociativeContainer`. The
`mapped_type` is either a pointer to a `vptr_type`, or a pointer to a
`vptr_type`.

### Members

#### register_vptrs

```c++
template<typename ForwardIterator>
auto register_vptrs(ForwardIterator first, ForwardIterator last) -> void;
```

Stores the pointers to v-tables in a _Map_.

#### dynamic_vptr

```c++
template<class Class>
auto dynamic_vptr(const Class& object) -> vptr_type;
```

Returns a pointer to the v-table for `object`.

If _Policy_ contains the `runtime_checks` facet, checks if _Class_ is
registered. If it is not, and if _Policy_ contains a `error_handler` facet,
calls its `error` function; then calls `abort`.
