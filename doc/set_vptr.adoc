
## set_vptr

### Synopsis

Defined in <boost/openmethod/core.hpp>.

```c++
namespace boost::openmethod {

template<
    class Class, class... Bases, class Policy = BOOST_OPENMETHOD_DEFAULT_POLICY
>
class set_vptr {
    set_vptr();
    ~set_vptr();
};

} // namespace boost::openmethod
```

### Description

`set_vptr` is a CRTP class template that manages a vptr in a class hierarchy.

If `Class` has no `Bases`, `set_vptr` adds a `boost_openmethod_vptr` private
member to `Class`. In either case, it sets the vptr to the v-table of `Class`
from `Policy`. It also creates a `boost_openmethod_vptr` friend function that
takes a a `const Class&` and returns the embedded vptr.

If `Class` has has more than one base, the `boost_openmethod_vptr` friend
function is also created, to prevent ambiguities. It returns one of the embedded
vptrs (it doesn't matter which one, as they all have the same value).

As part of its implementation, `set_vptr` may also declare one or two free functions (`boost_openmethod_policy` and `boost_openmethod_bases`).

### Members

#### constructor

```c++
set_vptr();
```

Casts `this` to `Class` and sets its `boost_openmethod_vptr` to the value of
`Policy::static_vptr<Class>`, or to its address if `Policy` contains
`indirect_vptr`.


#### destructor

```c++
~set_vptr();
```

Casts `this` to `Class` and sets its `boost_openmethod_vptr` to `nullptr`. As a
consequence, intrusive vptrs do not support passing an object being destroyed as
an argument to a method.
